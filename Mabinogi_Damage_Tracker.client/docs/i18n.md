# i18n Design (Client)

## Scope
- Applies to: `Mabinogi_Damage_Tracker.client/` only.
- Server is unchanged.
- Initial PR scope: a minimal subset of fixed UI strings (Settings + small nav labels).

## Library Choice
- `i18next` + `react-i18next`

## Resource Layout
- `src/localization/i18n/index.js` (i18n init)
- `src/localization/i18n/en.json`
- `src/localization/i18n/ja.json`

## Language Selection
Order:
1. If `localStorage.lang` exists, use it.
2. Else, if `navigator.language` starts with `ja`, use `ja`.
3. Otherwise use `en`.

- Default: `en`
- Fallback language: `en`
- Persist manual selection to `localStorage` key `lang`.

## Key Naming Rule
- Meaning-based keys (do NOT use English sentences as keys).
- Use screen/feature domain keys + `common.*` for shared concepts.
Examples:
- `common.loading`, `common.noData`
- `settings.title`, `settings.languageLabel`
- `sideMenu.settings`, `sideMenu.live`

## Anti-Key-Proliferation Rules
- Shared concepts must go under `common.*` and be reused across pages.
- Before adding a new key, search for an existing key with the same meaning.
- Do not create page-specific duplicates for the same concept (e.g., recording start/stop).
- `en.json` is the source of truth; other languages fall back to `en`.

## Formatting Policy
- Number/date formatting is unchanged in the initial PR.
- Chart translations are out-of-scope in the initial PR.

## Adding a New Translation
1. Add the key to `en.json` (required).
2. Add the Japanese translation to `ja.json` if the PR touches that UI.
3. Replace UI literals with `t("...")`.
4. Confirm you did not introduce duplicate keys for the same meaning.
